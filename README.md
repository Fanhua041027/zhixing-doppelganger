# 知行分身

基于意图识别的鸿蒙跨设备学习执行智能体。

## 项目简介

知行分身面向学习、竞赛备赛和项目推进场景。用户输入一个较大的学习目标后，系统会识别意图、检索本地个人上下文、生成任务卡片、分配设备角色，并给出可解释的评测指标。

当前版本采用本地确定性规则实现，优先保证演示稳定；架构上已经为后续接入检索增强、重排序、模型接口和鸿蒙系统能力预留边界。

## 已实现功能

- 学习目标输入与意图识别。
- 首页提供竞赛冲刺、考试复习、资料研读三类目标模板。
- 计划配置页支持填写截止时间、每日可用时间、当前难点和偏好设备。
- 首页展示个人约束摘要和今日行动清单。
- 智能体对话，包含检索依据和下一步动作。
- 执行计划生成与任务卡片展示。
- 手机、平板、穿戴设备的角色建模。
- 首页和任务页展示跨设备任务流转路径。
- 任务历史记录，用于说明分身会沉淀执行轨迹。
- 任务页支持单项勾选、执行推进和任务状态本地保存。
- 资料库支持文本粘贴导入、关键词检索，并接入对话依据。
- 资料库可根据导入资料生成可执行学习任务。
- 资料生成任务和复盘建议可以加入任务看板，并通过本地状态恢复展示。
- 本地首选项服务用于保存任务状态和导入资料，失败时自动降级到内存态。
- 个人画像信号、记忆、知识条目和冲突记录。
- 画像页根据复盘生成下一步建议，推动用户继续行动。
- 意图置信度、任务覆盖、设备角色等评测指标。
- 设置页中的检索增强、跨设备流转、严格上下文控制。
- 演示模式，用于稳定走完整答辩路径。
- 评审展示页和提交总览页，用于说明完成度、评分点和风险兜底。
- 日程和提醒等系统能力支持尝试拉起系统意图，失败时保留调用参数。

## 工程结构

- `entry/src/main/ets/pages`：鸿蒙界面页面。
- `entry/src/main/ets/models`：共享数据模型。
- `entry/src/main/ets/services`：智能体、评测、对话和系统能力服务。
- `entry/src/main/ets/repositories`：画像、知识和本地存储仓储。
- `docs`：开发计划、演示脚本、评分映射和提交检查清单。

## 演示路径

1. 打开首页。
2. 进入计划配置页，填写真实学习约束。
3. 回到首页刷新个人计划，查看个人约束和今日行动。
4. 点击“竞赛冲刺”目标模板或手动输入目标。
5. 展示识别意图、置信度、任务卡片和评测指标。
6. 打开对话页，询问 `如何降低演示风险？`，展示检索依据。
7. 打开任务页，点击“推进一次执行”，再查看复盘沉淀。
8. 打开画像页，展示个人画像、执行复盘、记忆、知识和冲突记录。
9. 打开评审页和提交总览页，确认评分点、提交资产和风险控制。

## 开发环境

- DevEco Studio 26.0.0 Beta1。
- HarmonyOS 应用。
- 设备类型：手机和平板。
- 语言：鸿蒙声明式开发语言。
