# 作品提交说明

## 作品名称

知行分身：基于意图识别的鸿蒙跨设备学习执行智能体。

## 问题

学生经常把学习目标分散在提醒、笔记和聊天记录里。普通助手可以回答问题，但很少能把意图转成持续可见的执行流程。

## 方案

知行分身识别学习或竞赛目标，检索本地个人上下文，生成任务卡片，分配设备角色，并展示可衡量的执行信号。第一版采用本地确定性规则，保证演示稳定。

## 创新点

- 意图到执行闭环：展示意图识别、任务拆解和下一步动作。
- 个人数字分身数据模型：画像、记忆、知识和冲突记录从一开始结构化。
- 检索增强预备架构：知识检索和打分隔离在服务层。
- 鸿蒙导向设计：任务流显式区分手机、平板和穿戴设备角色。
- 内置评测：置信度、任务覆盖和设备角色直接展示在界面中。
- 答辩兜底：评审页和提交总览页直接展示完成度、风险控制和提交材料。

## 已实现最小版本

- 首页：目标输入与执行计划生成。
- 计划配置页：填写截止时间、每日时间、当前难点和偏好设备。
- 首页：展示个人约束、今日行动和目标模板。
- 对话页：基于本地知识的智能体问答。
- 任务页：任务进度、任务卡片、设备分工、历史记录、单项勾选和本地状态恢复。
- 资料库页：文本导入、关键词检索和对话依据增强。
- 画像页：个人信号、执行复盘、记忆、知识和冲突记录。
- 设置页：智能体控制、日历调用准备和提醒调用准备。
- 评审页：评分点映射、完成度证据和答辩兜底。
- 提交总览页：提交检查、风险兜底和提交材料建议。
- 服务层：智能体规划、对话、知识检索、系统动作和评测。

## 下一步建设

- 扩展本地持久化覆盖画像、配置和复盘数据。
- 接入真实日历写入和提醒创建权限流程。
- 原型化手机和平板间的任务状态流转。
- 在智能体规划服务后接入真实模型和检索增强。

## 演示路径

1. 打开首页。
2. 输入 `三天内完成鸿蒙智能体竞赛演示`。
3. 生成执行计划。
4. 展示意图、跨设备任务流和评测指标。
5. 打开对话页，展示检索依据和下一步动作。
6. 打开任务页，推进一次执行，展示状态变化。
7. 打开画像页，展示执行复盘和数字分身数据模型。
8. 打开评审页，展示评分点映射和完成度证据。
9. 打开提交总览页，展示风险兜底和提交材料。
10. 打开设置页，展示系统能力扩展点。
